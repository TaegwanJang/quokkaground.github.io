포트폴리오 소개

  • 변호사가 만든 안전한 소개팅 앱 '지음'의 iOS·Android 앱과 백엔드(Node.js 서버·REST API), React 관리자 웹까지 전반을 개발·출시·운영한 풀스택 프로젝트입니다.
  • 안전한 만남을 지향하는 서비스 특성에 맞춰 본인인증과 가입 인증 플로우를 설계했습니다.

작업 범위

  • iOS(Swift)·Android(Kotlin) 앱 신규 개발, ReactorKit 상태관리 아키텍처
  • 본인인증 연동 및 가입 인증 플로우 설계
  • Node.js 기반 REST API 서버 설계·개발
  • 클라이언트-서버 데이터 모델 설계·연동
  • React 기반 관리자 웹(회원·매칭·운영 관리) 개발
  • Naver SENS 기반 SMS 발송 연동
  • 앱 출시·버전 관리·운영 중 성능 개선

주요 업무 및 기능

  • 앱: iOS·Android 신규 개발, ReactorKit 상태관리, 가입·본인인증 플로우
  • 서버: Node.js REST API 설계·개발, 데이터 모델·비즈니스 로직
  • 관리자: React 관리자 웹으로 회원·매칭·운영 관리
  • 연동: 본인인증, Naver SENS SMS
  • 운영: 출시·버전 관리·성능 개선

주안점

  • 앱·서버·관리자를 아우르는 데이터 모델 정합성과 API 설계
  • 안전한 서비스에 맞는 본인인증·가입 인증 흐름
  • 운영·확장을 고려한 상태관리(ReactorKit)와 관리자 운영 효율

문제점

  • 소개팅 서비스 특성상 안전(본인인증)과 신뢰가 핵심이지만, 이를 지탱할 인증·데이터 구조가 필요했습니다.
  • 앱, 서버, 관리자까지 하나의 데이터 모델로 정합성 있게 이어져야 했습니다.
  • 매칭·회원·운영을 관리자에서 효율적으로 다뤄야 했습니다.

프로젝트 목표

  • iOS·Android 앱과 Node.js 서버, React 관리자 웹을 하나의 구조로 개발·출시.
  • 본인인증·가입 인증 플로우로 안전한 가입 경험 제공.
  • 운영·확장을 고려한 API·데이터 모델과 관리자 기능 구축.

주안점

  • 앱-서버-관리자의 데이터 모델 정합성과 REST API 설계.
  • 본인인증·SMS 연동의 신뢰성.
  • 운영 효율을 높이는 관리자 웹과 성능 개선.

프로젝트 성과

iOS·Android 앱 신규 개발

Swift(iOS)·Kotlin(Android)로 소개팅 앱을 신규 개발하고, ReactorKit으로 상태관리 아키텍처를 구성했습니다.

Node.js 서버·REST API 개발

Node.js 기반 REST API 서버를 직접 설계·개발하고, 클라이언트-서버 데이터 모델을 설계·연동했습니다.

React 관리자 웹 구축

회원·매칭·운영을 관리하는 React 기반 관리자 웹을 개발했습니다.

본인인증·SMS 연동 및 출시

본인인증과 Naver SENS SMS를 연동하고, 앱을 출시해 운영·성능 개선까지 담당했습니다.
