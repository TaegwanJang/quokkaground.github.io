포트폴리오 소개

  • 운영 중인 사진인화 브랜드 앱 '포토몬'의 Android 버전을 Google Play 정책(Target API 35+)에 맞춰 Target SDK 34→35로 상향하고 안정적으로 배포한 유지보수 작업입니다.
  • 서비스 중단 없이 정책 대응·호환성 점검·검수를 거쳐 배포했습니다.

작업 범위

  • Android Target SDK 34 → 35 상향
  • Android 14/15 호환성 점검
  • 권한·백그라운드 실행 등 정책 변경 대응
  • Deprecated API 수정
  • Play Console 배포 검수 및 안정 배포

주요 업무 및 기능

  • 진단: Android 14/15에서의 동작 확인, Deprecated API 식별·수정
  • 정책 대응: 권한·백그라운드 정책 변경 대응, Target SDK 35 적용
  • 배포: Play Console 검수를 거친 운영 앱 안정 배포

주안점

  • 운영 중 앱을 중단·리스크 없이 상향·배포하는 것
  • Google Play 정책(API 35+) 요구사항 정확한 충족
  • 약속한 일정 준수와 진행상황 공유

문제점

  • Google Play가 Target API 35+ 미충족 앱의 업데이트를 제한해, 상향이 필요했습니다.
  • 운영 중인 앱이라 중단·오류 없이 안전하게 상향·배포해야 했습니다.
  • Android 14/15 정책 변경(권한·백그라운드·Deprecated API)에 대응해야 했습니다.

프로젝트 목표

  • Target SDK를 34→35로 상향해 Google Play 정책을 충족.
  • 호환성 점검·정책 대응·검수로 운영 앱을 안정 배포.
  • 약속한 일정 안에 완료.

주안점

  • 정책 요구사항의 정확한 충족.
  • 운영 앱 무중단 안정 배포.
  • 일정 준수·투명한 소통.

프로젝트 성과

Target SDK 34→35 상향·안정 배포

운영 중인 앱을 중단 없이 Target SDK 34에서 35로 상향하고 Play Console 검수를 거쳐 안정적으로 배포했습니다.

Google Play 정책·호환성 대응

Android 14/15 호환성을 점검하고, 권한·백그라운드 정책 변경과 Deprecated API를 수정해 정책을 충족했습니다.
