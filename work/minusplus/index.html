앱 개발

체성분 관리 앱 Flutter iOS·Android 개발 및 TUYA 체중계 연동

FlutterDartSwiftMethod ChannelEvent ChannelBLoCTUYA SDKBLE

참여 기간 · 2020.07 ~ 2020.10

포트폴리오 소개

  • TUYA 체중계를 활용해 체중·체성분을 관리하는 헬스케어 앱에서, TUYA 체중계 SDK를 Flutter 앱과 연동하는 부분을 담당했습니다.
  • TUYA SDK는 네이티브(iOS Swift 등) 형태라, 이를 Flutter 플러그인(Method/Event Channel)으로 만들어 Flutter 앱에서 체중계 데이터를 주고받도록 구현했습니다.

작업 범위

  • Flutter iOS/Android 앱 개발 (iOS 파트 Plugin은 Swift)
  • TUYA 체중계 SDK를 Flutter Plugin으로 직접 개발해 연동 (Method/Event Channel)
  • 체중계 측정 데이터의 앱 수신·전달 처리
  • BLoC 패턴 기반 상태관리 아키텍처 적용
  • 개발 완료 후 유지보수 대응

주요 업무 및 기능

  • Native ↔ Flutter 브릿지: Method/Event Channel로 네이티브 TUYA SDK를 Flutter에 연동
  • 하드웨어 연동: BLE 기반 체중계와의 측정 데이터 송수신을 Flutter 앱에서 처리
  • BLoC 아키텍처로 측정 데이터 흐름·상태 구조화
  • 배포 후 버그·정책 대응 등 유지보수

주안점

  • 네이티브 전용 하드웨어 SDK(TUYA)를 Flutter 플러그인으로 안정 연동
  • BLE 측정 데이터의 정확한 수신과 상태 동기화
  • 유지보수를 고려한 아키텍처(BLoC) 구조화

문제점

  • TUYA 체중계 SDK가 네이티브 형태라 Flutter 앱에 그대로 붙일 수 없었습니다.
  • BLE로 오가는 측정 데이터를 앱에서 안정적으로 수신·표시해야 했습니다.
  • 개발 이후 유지보수를 고려한 데이터 흐름·상태 구조가 필요했습니다.

프로젝트 목표

  • TUYA 체중계 SDK를 Flutter 플러그인으로 연동해 Flutter 앱에서 동작하도록 구현.
  • BLE 측정 데이터를 앱에서 정확히 수신·처리.
  • BLoC 아키텍처로 유지보수 가능한 구조 확보.

주안점

  • 네이티브 하드웨어 SDK의 Flutter 브릿지 안정성.
  • BLE 측정 데이터의 수신 정확성과 상태 동기화.
  • BLoC 기반 데이터 흐름 구조화.

프로젝트 성과

TUYA 체중계 SDK의 Flutter 연동

네이티브 TUYA 체중계 SDK를 Flutter 플러그인(Method/Event Channel)으로 연동해 앱에서 측정 데이터를 주고받도록 구현했습니다.

BLoC 기반 아키텍처 적용

BLoC 패턴으로 측정 데이터 흐름과 상태를 구조화하고, 개발 완료 이후 유지보수 단계까지 대응했습니다.
