포트폴리오 소개

  • 학습자와 튜터를 1:1로 잇는 교육 서비스 '랭디' 앱을 Flutter로 Android·iOS 동시 개발한 프로젝트입니다.
  • 외부 전화앱 없이 앱 안에서 실시간 음성 통화로 1:1 수업이 이뤄지도록 Tencent RTC 기반 통화(VoIP)를 구현했습니다.

작업 범위

  • Flutter 기반 Android·iOS 앱 동시 개발
  • Tencent RTC를 활용한 앱 내 1:1 통화(VoIP) 구현
  • Flutter Webview로 웹 콘텐츠·화면 연동
  • BLoC·GetX 기반 상태관리 아키텍처

주요 업무 및 기능

  • 앱: Flutter로 Android·iOS 동시 개발
  • 통화: Tencent RTC 기반 앱 내 실시간 음성 통화(1:1 수업)
  • 연동: Flutter Webview로 웹 화면·콘텐츠 연동
  • 아키텍처: BLoC·GetX 상태관리

주안점

  • 앱 이탈 없이 앱 안에서 이뤄지는 안정적인 1:1 실시간 통화
  • 하나의 Flutter 코드베이스로 Android·iOS 동시 대응
  • 상태관리(BLoC·GetX)와 Webview 연동의 일관성

문제점

  • 1:1 매칭 교육은 학습자와 튜터가 실시간으로 대화해야 하므로 안정적인 통화가 핵심이었습니다.
  • 외부 전화앱을 거치지 않고 앱 안에서 통화가 이뤄져야 학습 흐름이 끊기지 않습니다.
  • Android·iOS를 효율적으로 동시 대응해야 했습니다.

프로젝트 목표

  • Flutter로 Android·iOS 앱을 동시 개발.
  • Tencent RTC로 앱 내 1:1 실시간 음성 통화(VoIP)를 구현.
  • Webview 연동과 BLoC·GetX 상태관리로 일관된 구조 확보.

주안점

  • 앱 내 통화(VoIP)의 실시간성과 안정성.
  • 크로스플랫폼 동시 개발 효율.
  • 상태관리·Webview 연동의 일관성.

프로젝트 성과

Flutter Android·iOS 동시 개발

하나의 Flutter 코드베이스로 랭디 앱을 Android·iOS 동시 개발하고 BLoC·GetX로 상태관리를 구성했습니다.

앱 내 1:1 통화(VoIP) 구현

Tencent RTC를 활용해 외부 전화앱 없이 앱 안에서 학습자–튜터가 실시간 음성 통화로 1:1 수업하도록 구현했습니다.

Flutter Webview 연동

Flutter Webview로 웹 콘텐츠·화면을 앱에 연동해 학습 화면 구성을 유연하게 처리했습니다.
