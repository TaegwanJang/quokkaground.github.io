이멤버(이랜드 통합 멤버십) 결제·보안 모듈 및 이벤트 콘텐츠 개발

포트폴리오 소개

  • 이랜드 그룹 통합 멤버십 앱 '이멤버'에서 결제용 보안 모듈과 이벤트 콘텐츠를 개발하고, 네이티브 전용 보안 솔루션을 Flutter에 연동한 프로젝트입니다.
  • 앱 위변조 방지(AppIron)와 모바일 백신(mVaccine)은 iOS 네이티브(Swift) SDK 형태라, 이를 Flutter 플러그인(Method/Event Channel)으로 연동해 Flutter 앱에서 동작하도록 만드는 데 초점을 뒀습니다.

작업 범위 (제가 담당한 부분)

  • 결제용 보안 모듈 개발과 앱 내 결제(인앱결제) 결제 흐름
  • 바코드 적립·결제 화면
  • 네이티브 보안 SDK(AppIron 앱 위변조 방지, mVaccine 모바일 백신)의 Flutter 플러그인 연동
  • 이벤트 콘텐츠(룰렛·럭키드로우·이벤트 페이지) Flutter + GetX 개발
  • 완료 후 유지보수 대응

주요 업무 및 기능

  • Native ↔ Flutter 브릿지: Method Channel / Event Channel로 iOS 네이티브(Swift) 보안 SDK를 Flutter에 연동
  • 앱보안: AppIron 위변조 방지, mVaccine 백신 모듈을 Flutter 앱에서 정상 동작하도록 통합
  • 결제: 결제용 보안 모듈과 바코드 적립·결제 흐름·상태 처리
  • 이벤트 콘텐츠: 룰렛·럭키드로우 등 참여형 이벤트와 이벤트 페이지를 GetX 상태관리로 구현
  • 배포 후 운영: 버그·정책 대응 등 유지보수

주안점

  • 네이티브 전용 보안 SDK를 Flutter 플러그인으로 연동할 때의 브릿지 안정성
  • 결제와 보안이 함께 얽힌 흐름의 정확성과 상태 동기화
  • 참여형 이벤트 콘텐츠의 상태 관리와 안정성

문제점

  • 앱 위변조 방지(AppIron)와 모바일 백신(mVaccine)이 iOS 네이티브(Swift) SDK 형태라, Flutter로 개발된 앱에 그대로 붙이기 어려운 구조였습니다.
  • 멤버십 앱 특성상 결제와 보안이 함께 얽혀 있어, 연동이 불안정하면 리스크가 큰 영역이었습니다.
  • 참여형 이벤트는 보상·당첨 등 상태가 얽혀 있어, 콘텐츠라도 안정적인 상태 관리가 필요했습니다.

프로젝트 목표

  • 네이티브 전용 보안 SDK(AppIron·mVaccine)를 Flutter 플러그인으로 연동해 Flutter 앱에서 동작.
  • 결제용 보안 모듈과 결제 흐름을 안정적으로 개발.
  • 룰렛·럭키드로우 등 참여형 이벤트 콘텐츠를 GetX 기반으로 안정적으로 구현·운영.

주안점

  • 네이티브 보안 SDK(AppIron·mVaccine)의 Flutter 브릿지 안정성.
  • 결제 보안 모듈의 정확성과 결제 상태 동기화.
  • 참여형 이벤트의 상태 관리(보상·당첨 처리) 안정성.

프로젝트 성과

네이티브 보안 SDK의 Flutter 연동

앱 위변조 방지(AppIron)와 모바일 백신(mVaccine) 네이티브 SDK를 Flutter 플러그인으로 연동해 앱에서 정상 동작하도록 통합했습니다.

결제용 보안 모듈·바코드 결제 개발 완료

결제용 보안 모듈과 바코드 적립·결제 흐름을 개발 완료하고 배포 이후 유지보수 단계까지 안정적으로 운영했습니다.

참여형 이벤트 콘텐츠 개발·운영

룰렛·럭키드로우 등 참여형 이벤트 콘텐츠를 Flutter·GetX로 개발하고 보상·당첨 상태 처리와 운영을 담당했습니다.
